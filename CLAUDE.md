# British Birds Minecraft Mod

## What This Is

A Fabric mod for Minecraft Java Edition 26.1-rc-1 adding 100 British bird species as wild mobs with spawn eggs. Focus on biological accuracy: realistic models, appropriate biome spawning, species-specific flight and feeding behaviour, and authentic bird calls sourced from xeno-canto.

## Current Status

**All 5 pilot birds implemented and running in-game.** No crashes. Phases 1-8 complete.

### What's Working
- 5 birds: Robin, Blue Tit, Barn Owl, Peregrine Falcon, Mallard
- 3 locomotion archetypes: flying, water, ground (stub)
- 13 reusable AI goal classes
- Real bird sounds from xeno-canto (27 .ogg files, mostly CC BY-SA)
- 64x64 textures with species-accurate colours
- Higher-fidelity models with multiple cuboids per bird
- Creative tab with all spawn eggs
- Natural biome spawning
- Inter-species interactions (small birds flee raptors, peregrine hunts small birds)

### What Needs Work
- Textures are programmatic (tools/TextureGenerator.java) — could be hand-painted for more detail
- 95 more bird species to add (hierarchy is ready to scale)
- In-game visual tuning (animation smoothness, model proportions)
- 3 Robin sounds are CC BY-NC-SA (need BY-SA replacements for commercial use)

## Tech Stack

- **Minecraft:** Java Edition 26.1-rc-1 (unobfuscated — uses Mojang names, not Yarn)
- **Mod Loader:** Fabric (Loader 0.18.4, API 0.143.14+26.1)
- **Build:** Loom 1.15.5, Gradle 9.4.0, Java 25 (Adoptium Temurin)

- **Run command:** `JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-25.0.2.10-hotspot" ./gradlew runClient`

- **Textures:** 64x64 generated by `tools/TextureGenerator.java`
- **Sounds:** .ogg mono 44.1kHz Vorbis, sourced from xeno-canto

## MC 26.1 API Gotchas (IMPORTANT)

These class names differ from what documentation/tutorials suggest:

| Expected (Yarn/docs) | Actual (MC 26.1 Mojang) |
|---|---|
| `ResourceLocation` | `Identifier` (`net.minecraft.resources`) |
| `AnimalEntity` | `Animal` |
| `PathAwareEntity` | `PathfinderMob` |
| `MobSpawnType` | `EntitySpawnReason` |
| `CompoundTag` (NBT) | `ValueOutput`/`ValueInput` |
| `tag.getBoolean("key")` | `input.getBooleanOr("key", default)` |
| `EntityType.Builder.build(String)` | `.build(ResourceKey<EntityType<?>>)` |
| `SpawnEggItem(type, c1, c2, props)` | `SpawnEggItem(props.setId(key).spawnEgg(type))` |
| `EntityModelLayerRegistry` | `ModelLayerRegistry` |
| `Level.getDayTime()` | `Level.getOverworldClockTime()` |

Loom plugin ID: `net.fabricmc.fabric-loom` (NOT `net.fabric.fabric-loom`)


## Flight System Design

**Ground-first approach:** All birds use `GroundPathNavigation` by default. They walk/hop on the ground normally. Flight only happens when a specific flight goal activates.

**Flight goals control movement directly** via `setDeltaMovement()` — they don't use pathfinding for airborne movement. `setFlying(true)` calls `setNoGravity(true)`, `setFlying(false)` re-enables gravity.

**Only RaptorEntity** uses `FlyingPathNavigation` (since raptors spend most time airborne).

**Render state detects airborne:** `isFlying = entity.isFlying() || (!entity.onGround() && !entity.isInWater())` — ensures flying animation shows whenever a bird is off the ground, regardless of how it got there.
